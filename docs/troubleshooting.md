# 报错排查速查表

> 遇到报错别慌！对照下面的表格，找到你的问题，按步骤解决。

---

## 环境搭建

### "不是内部或外部命令"

**现象**：在终端输入 `node`、`npm` 或 `claude` 后，提示"不是内部或外部命令"。

**原因**：对应的软件没有正确安装，或者没有添加到系统的环境变量（PATH）中。

**解决**：
1. 重新按照[课前准备](/prepare)的步骤安装
2. 安装时注意勾选"Add to PATH"选项
3. 安装完成后**关闭终端再重新打开**（这一步很关键！）

### Node.js 安装失败

**现象**：下载或安装 Node.js 时报错，或安装后 `node --version` 没有输出。

**解决**：
1. 去 [Node.js 官网](https://nodejs.org/) 下载 LTS（长期支持）版本
2. 如果之前安装过旧版本，先卸载再重新安装
3. Windows 用户：以管理员身份运行安装程序

### npm install 报错

**现象**：运行 `npm install -g @anthropic-ai/claude-code` 时报错。

**解决**：
1. 确认 Node.js 已正确安装：运行 `node --version` 应显示版本号
2. 网络问题：检查网络连接，必要时尝试使用手机热点
3. 权限问题（Mac/Linux）：命令前加 `sudo`，即 `sudo npm install -g @anthropic-ai/claude-code`

### VS Code 打不开

**解决**：
1. 重启电脑后再试
2. 去 [VS Code 官网](https://code.visualstudio.com/) 下载最新版本重新安装
3. 如果是 Windows，检查是否被杀毒软件拦截

## 终端操作

### 终端在哪里打开？

在 VS Code 中按 `` Ctrl + ` ``（键盘左上角那个反引号键），或者点击菜单 **终端 → 新建终端**。

### 输入命令后没有任何反应

**可能原因**：
1. 命令还在执行中——有些操作需要等几秒钟
2. 命令拼写有误——仔细检查有没有打错字
3. 终端卡住了——试试按 `Ctrl + C` 中断，然后重新输入

### 提示"找不到文件"或"路径不存在"

**原因**：你当前所在的文件夹不对。

**解决**：
1. 在终端输入 `pwd`（Mac/Linux）或 `cd`（Windows）查看当前位置
2. 用 `cd 文件夹名` 切换到正确的项目文件夹
3. 也可以在 VS Code 中直接打开项目文件夹：**文件 → 打开文件夹**

## 网页相关

### 修改了代码，但浏览器没变化

**这是最常见的问题！** 解决方法：

1. **保存文件**：按 `Ctrl + S`（很多同学忘了这步！）
2. **刷新浏览器**：按 `Ctrl + R` 或 `F5`
3. **强制刷新**：按 `Ctrl + Shift + R`（清除缓存刷新）

::: tip 小贴士
养成习惯：改完代码 → `Ctrl + S` 保存 → 切到浏览器 → `Ctrl + R` 刷新。
:::

### CSS 样式不生效

**检查清单**：
1. CSS 代码是否写在 `<style>` 标签内？
2. 选择器名字是否和 HTML 中的一致？（注意大小写）
3. 是否缺少分号 `;` 或花括号 `{}`？
4. 试试强制刷新：`Ctrl + Shift + R`

### 图片不显示

**检查清单**：
1. 图片文件是否和 HTML 文件在同一个文件夹？
2. `<img>` 标签中的文件名是否拼写正确？（注意大小写和后缀名）
3. 图片格式是否正确？（常用：`.jpg`、`.png`、`.gif`）

### 网页在手机上看起来很乱

**原因**：没有添加响应式设计。

**解决**：把这个问题告诉 Claude Code——

::: info 魔法咒语
```
请帮我让这个网页在手机上也能正常显示，添加响应式设计
```
:::

## JavaScript 报错

### 按钮点了没反应

**检查清单**：
1. 打开浏览器的**开发者工具**（按 `F12`），查看红色错误信息
2. JavaScript 代码是否写在 `<script>` 标签内？
3. 按钮的 `onclick` 或事件绑定是否拼写正确？
4. 把错误信息复制给 Claude Code，让它帮你修复

::: tip 小贴士
`F12` 打开的开发者工具是你的好帮手！红色错误信息会告诉你哪一行代码出了问题。
:::

### 计时器不停 / 跑到负数

**原因**：计时器到 0 的时候没有正确停止。

**解决**：告诉 Claude Code 具体现象——
::: info 魔法咒语
```
我的番茄钟倒计时到0之后还在继续往负数走，请帮我修复，让它到0就停下来
```
:::

### 游戏翻牌后不翻回去 / 翻了超过2张

**原因**：游戏逻辑中的匹配检查有问题。

**解决**：描述清楚问题现象，让 Claude Code 帮你调试——
::: info 魔法咒语
```
我的翻牌游戏有个bug：我可以同时翻开3张以上的牌，但应该只能翻2张然后判断是否匹配，请帮我修复
```
:::

## Claude Code 相关

### Claude Code 没有反应

1. 检查网络连接是否正常
2. 在终端输入 `claude --version` 确认已安装
3. 试试输入 `/exit` 退出后重新运行 `claude`
4. 如果还不行，关闭终端重新打开

### 生成的代码有错误

这很正常！AI 也会犯错。你可以：
1. 把错误信息**直接粘贴**给 Claude Code，告诉它"报错了，帮我修一下"
2. 更详细地描述你想要的效果
3. 如果改了好几次还不对，试试让它重新开始写

### 效果和想象的不一样

AI 不是读心术，需要你说清楚。试试这样优化提示词：
- **描述具体**：不要只说"好看一点"，说"背景用浅蓝色，字体大一些，按钮是圆角的"
- **给参考**：说"我想要像XXX那样的效果"
- **分步骤**：一次只改一个地方，不要一口气提十个需求

## 万能解决思路

如果上面都没找到你的问题，试试这个通用流程：

1. **截图或复制**错误信息
2. **粘贴给 Claude Code**，问它"这个报错是什么意思？怎么修？"
3. 按照它的建议试试
4. 还不行？去 [GitHub Discussions](https://github.com/zhouning/beloved/discussions) 贴上截图提问

::: warning 记住
遇到 bug 不是你的错！这是编程中最正常不过的事情。每个程序员每天都在解决 bug，你正在学的就是最重要的技能——**解决问题的能力**。
:::
